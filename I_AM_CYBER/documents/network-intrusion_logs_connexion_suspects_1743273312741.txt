#### **Résumé du Scénario**  
Ce document présente les logs réseaux identifiés comme suspects suite à une tentative d'intrusion détectée sur le serveur principal de l'entreprise. L'analyse initiale a révélé plusieurs connexions provenant d'adresses IP externes suspectes, ainsi que des utilisations inhabituelles de protocoles sur des ports spécifiques.

#### **2. Détails supplémentaires**
- **Adresse IP interne suspecte :** `192.168.1.45`  
  - Cette adresse IP semble provenir d’un appareil interne compromis. Une analyse approfondie de l'équipement est recommandée.  
  - L'appareil tente d'accéder de manière répétée au port SSH (22) et au port RDP (3389).  
  - Liens avec des adresses IP externes non autorisées identifiées dans les logs.  

- **Adresse IP externe suspecte :** `203.0.113.45`  
  - Cette IP a envoyé plusieurs requêtes HTTP sur le port 8080, avec des payloads contenant des caractères typiques d’une injection SQL (`UNION SELECT`, `' OR 1=1 --`).  
  - Geo-localisation : Singapour.  

---

### **Analyse Technique**
1. **Tentative brute-force SSH**  
   - L’adresse interne `192.168.1.45` a initié **15 tentatives de connexion SSH** en moins de 3 minutes. Cela pourrait indiquer l’utilisation d’un outil de brute-force automatisé.  

2. **Injection SQL**  
   - L’IP externe `203.0.113.45` a tenté d’exploiter une vulnérabilité sur le serveur web interne. Les requêtes contiennent des paramètres exploitables.  

3. **Suspicion de Pivoting**  
   - Le comportement de l’IP interne `192.168.1.45` suggère une tentative de pivoting pour accéder au serveur RDP. Cela pourrait indiquer une compromission initiale.  

---

### **Actions Recommandées**
1. **Isolation**  
   - Déconnecter immédiatement l’appareil relié à l’adresse IP interne `192.168.1.45`.  

2. **Analyse approfondie**  
   - Procéder à une inspection complète des fichiers logs sur le serveur (SSH, RDP, HTTP).  
   - Scanner les équipements internes pour détecter la présence de malware.  

3. **Blocage des IP externes suspectes**  
   - Ajouter une règle de pare-feu pour bloquer les connexions des IP externes : `203.0.113.45` et `45.83.120.16`.  

4. **Audit des configurations réseau**  
   - Vérifier les ports ouverts sur le serveur et limiter les accès non indispensables.  
   - Renforcer les politiques d’authentification (ex. : 2FA sur SSH et RDP).
