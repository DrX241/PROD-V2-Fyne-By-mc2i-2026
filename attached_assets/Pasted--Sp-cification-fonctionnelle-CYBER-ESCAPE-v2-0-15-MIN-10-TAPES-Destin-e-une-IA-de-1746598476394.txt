# Spécification fonctionnelle – **CYBER ESCAPE v2.0 (15 MIN / 10 ÉTAPES)**

*(Destinée à une IA de génération de code sur **Replit**)*

---

## 1 · Objectif

Concevoir un serious‑game **100 % textuel** où le joueur dispose de **15 minutes (900 s)** pour franchir **10 étapes** successives liées aux grands piliers de la cybersécurité. Chaque étape :

* propose une mini‑mission ou un défi QCM ;
* attribue un **Jeton‑Clé** (clé cryptée, virus USB, badge IAM, etc.) ;
* donne un bonus ou déclenche une pénalité de temps ;
* peut impliquer un PNJ (dialogue interactif).

Le joueur sort s’il ouvre la « Porte Sigma » de l’étape 10 **et** répond correctement au quiz final. Des médailles (or/argent/bronze) sanctionnent la performance temps + connaissances.

---

## 2 · Boucle de jeu (commandes génériques)

| Commande                             | Effet                                     |
| ------------------------------------ | ----------------------------------------- |
| `aller <direction>`                  | Se déplacer N/S/E/O (labyrinthe textuel). |
| `regarder`                           | Décrit la salle, les PNJ, les portes.     |
| `parler <pnj>`                       | Ouvre dialogue (menu + saisie libre).     |
| `utiliser <objet>`                   | Emploie un item de l’inventaire.          |
| `déverrouiller <porte> [avec <clé>]` | Ouvre une porte verrouillée.              |
| `answer <A/B/C/D>`                   | Répond à un QCM.                          |
| `inventaire`                         | Affiche items.                            |

API interne (exemple Python) :

```python
game.start(player_id)
game.process(player_id, command_str) -> GameState
game.end(player_id)
```

---

## 3 · Chronomètre

* Temps initial : **900 s**.
* `update_timer(delta)` ajoute ou retire des secondes ; 0 ⇒ défaite.

## 4 · Inventaire initial

| ID               | Nom                 | Usage clé                                |
| ---------------- | ------------------- | ---------------------------------------- |
| `decrypt_device` | Décrypteur portable | Déchiffrer clés SHA‑1 / AES étapes 6 & 7 |
| `rfid_reader`    | Lecteur RFID        | Scanner badges IAM étape 3               |

Les autres objets sont gagnés : `virus_usb`, `cloud_token`, `supply_pass`, etc.

## 6 · Système de médailles

| Médaille   | Condition                               |
| ---------- | --------------------------------------- |
| **Or**     | ≥120 s restantes **et** ≥7/8 bonnes QCM |
| **Argent** | 30‑119 s **et** ≥6/8                    |
| **Bronze** | 1‑29 s **et** ≥5/8                      |
| **Échec**  | 0 s ou <5 QCM                           |

---

## 7 · PNJ génériques & IA dynamique

* **Echo** : mentor invisible ; après chaque étape, affiche un *mème ASCII* + indice.
* **Marchand de temps** (présent 2, 6, 9) : propose +60 s contre un item ou ‑90 s si refus.

## 8 · Fichiers & structure **Replit**

```

/game/data/
```

## 10 · Checklist pour l’IA développeuse

1. **Initialiser repo Replit** avec structure §8.
2. Importer `rich` et `prompt_toolkit` pour I/O stylé.
3. Implémenter `GameState` (timer, position, inventaire).
4. Charger `steps.json`, générer salles & PNJ.
5. Gérer `update_timer()` sur chaque action.
6. Scripts des 10 étapes selon table §5.
7. Dialogues PNJ via fonctions dans `game/dialogue.py`.
8. QCM engine (`evaluate_answer`).
9. Log JSON dans Replit DB.
10. Tests unitaires sur timer & médailles.
