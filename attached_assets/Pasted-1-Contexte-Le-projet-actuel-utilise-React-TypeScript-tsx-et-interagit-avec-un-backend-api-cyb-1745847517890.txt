1. Contexte
Le projet actuel utilise React, TypeScript (.tsx), et interagit avec un backend /api/cyber/chat pour un agent conversationnel de cybersécurité.

Nous souhaitons maintenant créer un nouveau module indépendant, beaucoup plus immersif, réaliste, gamifié, et orienté gestion de crise.

Ce module ne remplace pas l’actuel, il est ajouté en parallèle dans un dossier séparé.

2. Objectif du nouveau module
Créer CyberCrisisChallenge :
Un simulateur de gestion de crise cybersécurité réaliste, où l’utilisateur :

Choisit un rôle professionnel.

Vit une crise cyber (ransomware, fuite de données, piratage, etc.).

Interagit avec plusieurs personnalités IA (DG stressé, Hacker provocateur, employés paniqués, etc.).

Écrit librement ses décisions (pas juste des QCM).

Gère un budget de crise de 400 000€.

Accumule des points en fonction de la pertinence de ses décisions.

Voit l’impact de ses choix : aggravation ou amélioration de la situation.

3. Ce que vous devez créer
📁 Dossier : src/modules/cyber-crisis
Contenu du dossier :

a) CyberCrisisContext.tsx
Un nouveau Provider React qui gère :

Prénom de l'utilisateur.

Rôle choisi (parmi une liste proposée).

Scénario de crise en cours.

Liste des interlocuteurs simulés (DG, DPO, Hacker, Employés...).

Budget restant.

Score accumulé.

Historique des messages.

Fonctions : envoyer une réponse, recevoir une réponse IA, impact budget et score, changer d'étape.

b) CrisisChatPage.tsx
Une page de chat immersive :

Style pro, réaliste (simuler des emails, SMS, alertes dans l'interface).

Saisie libre de texte par l’utilisateur.

Messages des interlocuteurs IA simulés.

Indicateurs visuels : budget restant, score, niveau d'alerte.

c) ScenarioList.ts
Fichier TypeScript contenant des scénarios prédéfinis :

Exemple de scénarios :

Rançongiciel sur serveurs de production.

Piratage de messagerie interne.

Fuite massive de données RH.

Chaque scénario contient :

Description de départ.

Liste des événements possibles selon les choix.

Coût potentiel pour chaque décision.

d) PersonalityProfiles.ts
Fichier TypeScript contenant la base des profils IA :

Exemples de profils :

Nosing (DG autoritaire et impatient).

Edouard (Directeur Financier soucieux du budget).

Eddy (Expert IA, propose des solutions techniques).

Neil (RSSI, prudent).

Le Hacker (ironique et provocateur).

Chaque profil doit définir :

Ton de voix.

Réactions types.

Tolérance au risque.

e) API Backend /api/cyber/crisis-chat
Créer une nouvelle route serveur qui utilise Azure OpenAI.

Dans cette route :

Prendre en compte :

Le scénario sélectionné.

Le rôle de l’utilisateur.

L’historique de la conversation.

Utiliser un prompt système spécifique (fournit ci-dessous).

4. Prompt système pour /api/cyber/crisis-chat
text
Copier
Modifier
Tu es un simulateur de gestion de crise cyber en entreprise.
Ton rôle est de simuler une situation d'attaque critique en cybersécurité.

Instructions :
- Crée des dialogues réalistes avec différents interlocuteurs (DG, DPO, Hacker, Employés...).
- Chaque action de l'utilisateur doit entraîner une réaction logique de la situation.
- Fais ressentir la pression : stress, budget qui fond, critiques du DG, provocations du Hacker.
- À chaque message, propose un nouveau dilemme ou événement inattendu si l'utilisateur gère mal.
- Réponds toujours en français.
- Sois réaliste : certaines décisions doivent avoir des conséquences négatives immédiates.
- Utilise un style court, clair, professionnel.
- Si l'utilisateur dépasse le budget ou prend trop de mauvaises décisions, aggrave la situation.
- À la fin du scénario, prépare un bilan chiffré (coût total, réputation, satisfaction DG).
- N'utilise jamais de ton neutre : sois vivant, émotionnel, humain.
5. Flux attendu dans l'application

Étape	Action utilisateur	Réponse IA
1	Saisie du prénom	Accueil personnalisé
2	Choix du rôle (ex: RSSI, DG, DPO, etc.)	Confirmation du rôle
3	Présentation du scénario choisi	Démarrage de la simulation
4	Dialogue libre	Impact immédiat sur score/budget/crise
5	Enchaînement des événements	Selon choix utilisateur
6	Fin de simulation	Bilan complet avec note finale
6. Contraintes importantes
Ne pas modifier les autres modules existants (ChatContext.tsx, etc.).

Créer un module totalement séparé pour éviter toute régression.

Utiliser uniquement React, TypeScript, Tailwind CSS si besoin, et Azure OpenAI comme modèle backend.

Structure de code claire et évolutive car d’autres scénarios et fonctionnalités seront ajoutées plus tard.
